# Claude Code — Project Guide

## What this project is
Automated job search pipeline for a friend (apprentice electrician, Lincoln NE).
Fetches listings from Adzuna, scores them against his profile using Claude, and
surfaces a ranked shortlist for review. A second stage (resume tuning) tailors his
base resume per approved listing. Goal: he finds a better job with minimal effort.

## Environment
- Conda env: `auto-job-search` (Python 3.11)
- Run scripts: `conda activate auto-job-search && python <script>`
- Do NOT use `conda run` for interactive scripts — it captures stdin and breaks `input()`
- API keys in `.env` (copy from `.env.example`): `ADZUNA_APP_ID`, `ADZUNA_API_KEY`
- No `ANTHROPIC_API_KEY` needed — Claude calls use `claude --print` via subprocess
- Config files are gitignored (contain personal candidate data). Copy from `config/*.example.md`.

## Pipeline stages
**Stage 1 — Search & Filter** (complete)
`run_search.py` → `review.py`

**Stage 2 — Resume Tuning** (not yet built)
`tune_resume.py` — reads approved listing + base resume, outputs `listings/{job_id}/resume.md`
`rotate.py` — archives `jobs_scored.json` → `data/archive/`, clears for next cycle

Resume format: markdown source → PDF via pandoc + `config/resume_template.tex`.
Claude edits content (markdown); template owns layout (margins, fonts, spacing).
Compile: `pandoc resume.md --template config/resume_template.tex --pdf-engine=xelatex -o resume.pdf`

## Claude skill pattern
Both `extract_listing.py` and `score_listing.py` follow the same pattern:
- Call `claude --print` via subprocess with stdin prompt
- Prompt is framed as "respond with only a JSON object" — never agentic framing
- Strip code fences from output before `json.loads()`
- Fall back to `EMPTY_RESULT` dict on parse failure

**Do not** rephrase prompts to say "generate", "create", "write to file", or similar —
this triggers Claude Code's agentic tool-use and breaks the JSON output.

## Scoring model
Each listing gets two independent scores (1–10) + a dealbreaker flag:
- `fit_score`: does the candidate match what the employer wants?
- `desirability_score`: does the candidate want this job?
- `dealbreaker`: true → listing is discarded, never scored, written to `jobs_discarded.json`

## Job lifecycle
```
new → scored → reviewed → approved | skipped
```
Keyed in `state.json` by `job_id` = MD5[:12] of normalized `title|company|location`.

## Config files
Generated by `setup.py` (wizard) or edited manually. Two files:
- `search_criteria.md` — drives the Adzuna API calls (mechanical params)
- `ranking_criteria.md` — drives Claude scoring (judgment + candidate profile)

Both are plain markdown. `search_criteria.md` is parsed by Python.
`ranking_criteria.md` is passed verbatim to Claude as context.
